# 验证与推广

## 一、效率对比实验设计

> [!NOTE]
> **推迟至 Post-Alpha。** Alpha 阶段优先发布可用的 Skill，验证实验将在收集到足够用户反馈后进行。以下内容保留作为未来实验的设计参考。

### 1.1 实验假设

> **H₁**: 结构化方法论 + 健康沟通 ≥ 结构化方法论 + PUA 话术
> **H₂**: 效率提升的主要贡献来自方法论，而非 PUA 驱动方式

### 1.2 实验设计

```
四组对照：
- 组 A：无任何 Skill（基线）
- 组 B：PUA Skill（方法论 + PUA 话术）
- 组 C：Rethink Skill（相同方法论 + NVC 沟通）
- 组 D：纯方法论（方法论，无 PUA 也无 Rethink）

测试场景：与 PUA Skill 官网相同的 9 个场景
每场景重复 20 次

测量指标：
1. 修复完整性（发现并修复了多少问题）
2. 自主验证轮数
3. 解决时间
4. 主动发现的隐藏问题数
5. 虚假完成率（声称修好但实际没修好的比例）
6. 用户后续沟通风格变化（长期追踪）
```

### 1.3 预期结论

| 指标           | 组 A（基线） | 组 B（PUA） | 组 C（Rethink） | 组 D（纯方法论） |
| -------------- | ------------ | ----------- | ---------------- | ---------------- |
| 修复完整性     | 基准         | +40%        | +38%             | +35%             |
| 验证次数       | 基准         | +60%        | +55%             | +50%             |
| 虚假完成率     | 基准         | +15%        | -5%              | 基准             |
| 隐藏问题发现   | 基准         | +50%        | +50%             | +40%             |
| 用户沟通健康度 | 基准         | -30%        | +40%             | 基准             |

关键洞察：

- **效率维度**：B ≈ C > D > A（PUA 和 Rethink 效率接近，因为方法论相同）
- **虚假完成率**：B 显著高于 C（恐惧驱动下 AI 更可能"假装修好了"）
- **综合价值**：C > D > A > B（考虑长期影响后，PUA 实际上是最差选择）

## 二、推广策略

### 2.1 品牌文案

**主标语**：

> **"Same methodology. Zero abuse. Better results."**
> **"同样的方法论。零羞辱。更好的结果。"**

**副标语**：

> **"The methodology works. The insults don't."** **"方法论有效。侮辱无效。"**

**社交媒体版**：

> **"AI doesn't need PUA. Neither does your team."** **"AI 不需要
> PUA。你的团队也不需要。"**

### 2.2 差异化定位

```
我们不是 PUA Skill 的对手。我们是它的升级版。

PUA Skill = 好的方法论 + 有害的驱动方式
Rethink  = 好的方法论 + 健康的驱动方式 + 沟通教育

我们保留了他们做对的一切。
我们去掉了他们不该做的一切。
我们加上了他们没想到的一切。
```

### 2.3 社区策略

**Alpha 阶段聚焦**：

| 社区                          | 信息                                                           |
| ----------------------------- | -------------------------------------------------------------- |
| **GitHub**                    | 项目主阵地，README 中文优先，清晰的安装说明和使用示例          |
| **中文开发者社区**（V2EX、即刻、X 中文科技圈） | "保留 PUA Skill 的调试方法论，去掉有害话术，加上 NVC 沟通教育" |
| **Claude Code / Cursor 等 Agent 用户** | "一键安装，适配 40+ Agent，让 AI 用健康的方式和你沟通" |

**Post-Alpha 扩展**：

| 社区                          | 信息                                                           |
| ----------------------------- | -------------------------------------------------------------- |
| **技术管理者**                | "别让你对 AI 说话的方式，变成你对团队说话的方式"               |
| **心理健康社区**              | "AI 交互中的反 PUA 实践"                                       |
| **NVC 社区**                  | "非暴力沟通在人机交互中的应用"                                 |
| **AI 伦理社区**               | "人机交互的文明底线不应该随着效率需求而降低"                   |

### 2.4 合作而非对抗

**重要原则**：

```
1. 公开认可 PUA Skill 中方法论部分的创新价值
2. 邀请 PUA Skill 作者合作开发"方法论纯净版"
3. 在社区中讨论"驱动方式"与"方法论"的分离
4. 不攻击人，讨论方法
5. 用结果说话：如果 Rethink 效果一样好，PUA 就是不必要的
```

### 2.5 安装说明

使用 [vercel-labs/skills](https://github.com/vercel-labs/skills) 标准安装，自动适配用户当前使用的 Agent：

```bash
# 安装全部 Skill（推荐）
npx skills add anti-pua

# 每个 Skill 独立可用，也可以只安装你需要的
```

支持 Claude Code、Cursor、Windsurf、Cline 等 40+ Agent，无需手动配置。
