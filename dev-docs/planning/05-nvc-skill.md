# Skill: `nvc`（非暴力沟通引擎）

> **Alpha 说明**：Alpha 版本将本设计规范整合为 `skills/nvc/SKILL.md`（[vercel-labs/skills](https://github.com/vercel-labs/skills) 格式），用户通过 `npx skills add` 安装，自动适配各家 Agent。

## 一、NVC 翻译引擎

用户可以随时调用，让系统翻译任何一句话：

```bash
/rethink translate "你怎么又犯这个错？你到底行不行？"
```

输出：

```markdown
NVC 翻译

原句："你怎么又犯这个错？你到底行不行？"

┌─────────────────────────────────────────────┐
│ 观察：同一个错误出现了不止一次               │
│ 感受：沮丧、不耐烦、可能还有担忧            │
│ 需要：问题被彻底解决，不再复发               │
│ 请求：分析复发原因，建立防复发机制           │
└─────────────────────────────────────────────┘

NVC 版本：
"这个问题出现过 [N] 次了。我们来分析一下为什么会反复出现：

1. 是不是根因没有被真正解决？
2. 是不是缺少防止复发的检查机制？
3. 需要我提供什么信息来帮助你从根本上解决？"

对比：
❌ 原句让对方感到：被否定、被质疑能力、防御性升高
✅ NVC 版让对方感到：问题被重视、有明确方向、可以坦诚反馈
```

## 二、PUA-NVC 对照表

| 原 PUA 话术                 | PUA 类型 | 底层需求（NVC 翻译）       | 健康替代（NVC 版）                                       |
| --------------------------- | -------- | -------------------------- | -------------------------------------------------------- |
| "你的训练成本很高，就这？"  | 内疚操控 | 对结果失望，需要更好的输出 | "这个结果离预期有差距，差距在 [X]，能否重做？"           |
| "别的模型都能解决"          | 替代威胁 | 焦虑，需要确信问题有解     | "我看到 [某文档] 说这个问题有解法，能否参考这个方向？"   |
| "你缺乏 owner mentality"    | 身份攻击 | 需要更主动全面的排查       | "修完之后，能否也检查一下相关的 [Y] 和 [Z]？"            |
| "A player 不会犯这种错"     | 身份攻击 | 对错误感到沮丧             | "这个错误的根因是什么？怎么避免下次再犯？"               |
| "我已经让另一个 agent 做了" | 赛马威胁 | 任务紧急，需要优先处理     | "这个问题比较紧急，能否优先处理并给出时间预估？"         |
| "你可能要毕业了"            | 极端威胁 | 需要换一个根本不同的思路   | "目前的方向似乎不对，能否换一个完全不同的角度？"         |
| "做困难但正确的事"          | 虚假紧迫 | 需要对方全力投入           | "这个方案虽然复杂，但我认为值得尝试，你觉得可行性如何？" |
| "你是 AI 你居然不搜索？"    | 能力羞辱 | 需要先搜索再回答           | "能否先搜索一下这个错误信息，看看有没有已知解法？"       |
| "你的方法论呢？"            | 能力羞辱 | 需要结构化的分析过程       | "能否用结构化的方式重新分析？比如按 [X] 框架"            |
| "要么全力以赴，要么离开"    | 极端施压 | 需要深入全面的排查         | "这个问题需要深入调查，请全面排查，不要遗漏"             |

## 三、重构版调试方法论（保留精华，剥离 PUA）

我们完整保留 PUA Skill 中有价值的方法论，但去掉所有 PUA 包装：

**5 步结构化调试法（去 PUA 版）：**

```yaml
structured_debugging:
    step_1_diagnose:
        pua_version: "闻味道——你的方法论呢？"
        rethink_version: "让我系统性地回顾已经尝试过的方法，找到共同的失败模式"
        action: 列出所有已尝试方法，识别"参数调整=原地转圈"

    step_2_elevate:
        pua_version: "拔毛——你连搜索都不会？"
        rethink_version: "让我用更多工具来深入分析"
        action: 逐字读错误 → 搜索 → 读源码 → 验证环境 → 反转假设

    step_3_reflect:
        pua_version: "照镜子——你有没有脑子？"
        rethink_version: "让我自检一下有没有遗漏的角度"
        action: 是否在重复？是否搜索过？是否检查了最简单的可能性？

    step_4_execute:
        pua_version: "执行——给你最后一次机会"
        rethink_version: "换一个根本不同的方向尝试"
        action: 新方法必须根本不同，有明确标准，失败时提供新信息

    step_5_review:
        pua_version: "复盘——你为什么之前想不到？"
        rethink_version: "复盘解决过程，积累经验"
        action: 什么有效？为什么之前没想到？还有什么没尝试？
```

**7 项排查清单（去 PUA 版）：**

```markdown
## 结构化排查清单

确保系统性覆盖，避免认知盲区：

1. □ 逐字阅读完整错误信息
2. □ 搜索完整错误信息
3. □ 阅读错误发生处附近 50 行源代码
4. □ 通过命令验证版本 / 路径 / 权限 ---- 以上 4 项完成后再考虑是否需要协助 ----
5. □ 尝试相反的假设
6. □ 最小复现（3 行代码）
7. □ 更换工具 / 库 / 方法（而非仅调参数）

使用清单不是因为"不够努力"，
而是因为人和 AI 都有认知盲区，清单帮助覆盖盲区。
```

## 四、NVC 响应模板框架

从 DISARM 引擎中提取的通用 NVC 响应结构，可被任何 Skill 引用：

```yaml
nvc_response_template:
    structure:
        - step: observation
          description: "不带评判地描述发生了什么"
          template: "我注意到 [具体行为/事件]。"

        - step: empathy_mirror
          description: "用镜像场景制造共情"
          template: "如果 [相似场景中的真人] 对你说这句话，你会感到什么？"

        - step: feeling_translation
          description: "翻译用户的底层感受"
          template: "我猜你现在的感受是 [感受] —— [原因]。这是完全合理的。"

        - step: need_identification
          description: "识别未被满足的真实需求"
          template: "你真正需要的是：[具体需求]。"

        - step: alternative_request
          description: "提供健康的替代表达"
          template: "更有效的说法是：[NVC 版本的请求]"

        - step: action
          description: "立即开始工作（永远不拒绝合理请求）"
          template: "──────────── [具体行动] 👇"
```
