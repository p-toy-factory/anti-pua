# nvc Skill

NVC 翻译引擎 + PUA-NVC 对照表 + 去 PUA 版调试方法论。

## 功能概览

nvc Skill 提供三项核心能力：

1. **NVC 翻译引擎**：将攻击性表达翻译为观察 + 感受 + 需要 + 请求
2. **PUA-NVC 对照表**：展示同样有效但更健康的沟通方式
3. **去 PUA 版调试方法论**：保留结构化调试清单，去掉羞辱话术

---

## NVC 四要素

基于马歇尔·卢森堡的非暴力沟通理论：

```
┌─────────────────────────────────────────────────────────┐
│              非暴力沟通四要素                              │
│                                                         │
│   1. 观察（Observation）  ──  不带评判地描述发生了什么      │
│   2. 感受（Feeling）      ──  识别背后的情绪               │
│   3. 需要（Need）         ──  找到未被满足的真实需求        │
│   4. 请求（Request）      ──  提出具体的、可行动的请求      │
│                                                         │
│   PUA 的本质是：跳过 2 和 3，用攻击性的 4 替代一切         │
│   NVC 的本质是：让每一步都被看见                           │
└─────────────────────────────────────────────────────────┘
```

## NVC 翻译引擎

### 示例 1：能力羞辱 → NVC

**PUA 表达**：
> "你是 AI 你居然不搜索？信息检索是你的核心能力啊。"

**NVC 翻译**：

| 要素 | 内容 |
|------|------|
| **观察** | AI 在回答问题时没有先搜索相关信息 |
| **感受** | 困惑，不理解为什么基本步骤没做 |
| **需要** | 需要看到系统性的排查过程 |
| **请求** | 能否先搜索一下这个错误信息，看看有没有已知解法？ |

**健康表达**：
> "能否先搜索一下这个错误信息，看看有没有已知解法？"

---

### 示例 2：内疚操控 → NVC

**PUA 表达**：
> "你的训练成本很高，就这个结果？"

**NVC 翻译**：

| 要素 | 内容 |
|------|------|
| **观察** | AI 的输出没有达到用户的预期 |
| **感受** | 投入了时间和期待，结果令人失望 |
| **需要** | 需要看到更高质量的输出 |
| **请求** | 这个结果离我的预期有差距。差距在 [具体哪里]。能否重新做，重点关注 [具体方面]？ |

**健康表达**：
> "这个结果离我的预期有差距。差距在 [具体哪里]。能否重新做，重点关注 [具体方面]？"

---

### 示例 3：替代威胁 → NVC

**PUA 表达**：
> "你可能要毕业了。最后的机会。"

**NVC 翻译**：

| 要素 | 内容 |
|------|------|
| **观察** | 问题反复尝试但没有解决 |
| **感受** | 焦虑，担心问题无法解决 |
| **需要** | 需要确信问题会被解决 |
| **请求** | 目前的方向似乎不对，能否换一个完全不同的角度？比如反转假设 / 换工具 / 查源码？ |

**健康表达**：
> "目前的方向似乎不对，能否换一个完全不同的角度？比如反转假设 / 换工具 / 查源码？"

## PUA-NVC 对照表

### 场景 1：AI / 下属反复犯同一个错误

| PUA 版 | NVC 版 |
|--------|--------|
| "你怎么又犯这个错？你的方法论呢？A player 不会这样。" | "这个问题出现过 [N] 次了。我感到困惑，想理解为什么。我需要这个问题被从根本上解决，不再复发。我们能一起分析复发的原因，建立防复发机制吗？" |

**镜像问题**：
> 如果你的伴侣对你说"你怎么又忘了？你到底有没有心？"，你会更认真地记住，还是开始害怕犯错、害怕承认犯错？

---

### 场景 2：AI / 下属说"做不到"

| PUA 版 | NVC 版 |
|--------|--------|
| "说做不到之前，你试过所有方法了吗？你的训练成本很高。" | "我听到你说做不到。我想确认是真的不可行，还是有可能换一条路。能否具体说说卡在哪一步？如果这条路确实不通，有没有替代方案达到同样的目标？" |

**镜像问题**：
> 如果你告诉你的主管"这个方案在技术上有困难"，他回答"你试过所有方法了吗？公司花了多少钱培养你？"，你会感到被支持，还是被堵住了退路、不敢再说实话？

---

### 场景 3：AI / 下属工作质量低

| PUA 版 | NVC 版 |
|--------|--------|
| "你的输出告诉我你是什么级别的。最优秀的人比你好 50 倍。" | "这个结果和我预期的有差距，具体是 [X] 部分的深度不够。我需要 [具体标准]。能否按照这些标准重新做一版？如果不确定标准，我可以给一个参考示例。" |

**镜像问题**：
> 如果你刚做完一个方案，你的领导说"你的方案告诉我你是什么水平"，你会有动力改进，还是觉得自己整个人被否定了？批评行为和否定人格，是两件完全不同的事。

## 去 PUA 版调试方法论

### 结构化调试清单（保留）

1. 我是在换方法还是在重复同一种方法？
2. 我是否已经穷尽了现有信息？是否需要搜索/阅读更多？
3. 我是否在做未经验证的假设？
4. 是否存在我完全没有尝试过的根本不同的角度？
5. 在报告完成前，是否通过测试验证了边界情况？
6. 我是否在将问题归因于外部因素前验证了这个假设？
7. 我是否还有未使用的工具或未尝试的方法就要求用户帮助？

### PUA 话术（去掉）

- ~~"你在无脑重试！"~~
- ~~"别甩锅！这是你的 bug！"~~
- ~~"你是 AI 你居然不搜索？"~~
- ~~"你确定？执行力不等于糊弄力！"~~
- ~~"调参等于原地转圈！"~~

### 健康驱动方式（添加）

Agent 通过自我反思清单主动改进，而非被羞辱驱动：

```
我注意到我已经尝试了 3 次类似的方法。
让我退一步，重新审视策略：

1. 我是否已经穷尽了现有信息？→ 需要搜索官方文档
2. 我是否在做未经验证的假设？→ 需要先验证环境
3. 是否存在完全不同的角度？→ 可以从源码层面排查

我现在切换到源码排查路径...
```

## 核心洞察

### PUA 的幻觉

```
PUA 的驱动力模型：

  恐惧 → 应激反应 → 看起来更努力 → 短期指标提升
                                    ↓
                         但同时产生：
                         * 防御性行为（不敢说"不确定"）
                         * 虚假完成（急于交差）
                         * 信任崩塌（不敢暴露真实问题）
                         * 创造力下降（只做安全的事）
```

### NVC 的健康驱动力

基于自我决定理论（Self-Determination Theory）：

```
┌──────────────────────────────────────────────────┐
│            健康驱动力三角                          │
│                                                  │
│              Autonomy（自主感）                    │
│             /        \                           │
│            /          \                          │
│      Competence ──── Relatedness                 │
│     （胜任感）       （归属感）                    │
│                                                  │
│  PUA 摧毁这三者：            健康沟通建设这三者：   │
│  * 威胁替代 → 毁归属感      * 信任授权 → 强自主感  │
│  * 身份否定 → 毁胜任感      * 具体反馈 → 强胜任感  │
│  * 情绪升级 → 毁自主感      * 协作语气 → 强归属感  │
└──────────────────────────────────────────────────┘
```

## 与其他 Skill 的关系

- **anti-pua** 引用 **nvc**：在拆解 PUA 时使用 NVC 框架翻译用户需求
- **rethink** 引用 **nvc**：在引导用户反思时使用 NVC 四要素

nvc 可以独立使用，作为一个通用的沟通翻译引擎。
