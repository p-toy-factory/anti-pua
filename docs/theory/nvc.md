# NVC 四要素入门

非暴力沟通（Nonviolent Communication, NVC）是马歇尔·卢森堡博士开发的一套沟通框架。

## 四要素结构

```
┌─────────────────────────────────────────────────────────┐
│              非暴力沟通四要素                              │
│                                                         │
│   1. 观察（Observation）  ──  不带评判地描述发生了什么      │
│   2. 感受（Feeling）      ──  识别背后的情绪               │
│   3. 需要（Need）         ──  找到未被满足的真实需求        │
│   4. 请求（Request）      ──  提出具体的、可行动的请求      │
│                                                         │
│   PUA 的本质是：跳过 2 和 3，用攻击性的 4 替代一切         │
│   NVC 的本质是：让每一步都被看见                           │
└─────────────────────────────────────────────────────────┘
```

---

## 1. 观察（Observation）

**定义**：不带评判地描述发生了什么。

**区别**：

| 评判性表达 | 观察性表达 |
|------------|------------|
| "你总是不认真" | "这是第三次出现同样的错误" |
| "你的水平很差" | "这个输出没有达到我设定的标准" |
| "你在敷衍我" | "你用了 10 秒就说完成了，但没有验证" |

**核心要点**：把你看到的事实和你的解读分开。

---

## 2. 感受（Feeling）

**定义**：识别背后的情绪。

**真实感受 vs 假感受**：

| 真实感受（你的情绪） | 假感受（你对对方的评判） |
|----------------------|--------------------------|
| "我感到沮丧" | "我感到你不尊重我" |
| "我感到焦虑" | "我感到你在敷衍" |
| "我感到失望" | "我感到你没有努力" |

**核心要点**：真实感受是你内在的情绪状态，假感受是把评判包装成感受。

---

## 3. 需要（Need）

**定义**：找到未被满足的真实需求。

**需要 vs 策略**：

| 需要（抽象的、普遍的） | 策略（具体的、个人的） |
|------------------------|------------------------|
| "我需要问题被解决" | "我需要你重新做" |
| "我需要被理解" | "我需要你多花时间听我说" |
| "我需要确定性" | "我需要你给我一个承诺" |

**核心要点**：需要是所有人都能理解的普遍价值（安全感、自主感、被尊重、效率等），策略是满足需要的一种具体方式，但不是唯一方式。

---

## 4. 请求（Request）

**定义**：提出具体的、可行动的请求。

**请求 vs 要求**：

| 请求（开放的） | 要求（强迫的） |
|----------------|----------------|
| "能否先搜索一下？" | "你必须先搜索" |
| "我们能换个方法试试吗？" | "换个方法！" |
| "你愿意重新做一版吗？" | "给我重做！" |

**核心要点**：请求给对方说"不"的空间，要求不给。请求是邀请，要求是命令。

## NVC 应用于 Anti-PUA

### 示例：用户说"你的训练成本很高，就这？"

**拆解为 NVC 四要素**：

| 要素 | 解读 |
|------|------|
| **观察** | AI 的输出没有达到用户的预期 |
| **感受** | 用户感到沮丧、焦虑、可能还有时间压力 |
| **需要** | 用户需要问题被彻底解决，需要确信 AI 在认真对待这个任务 |
| **请求** | 用户希望 AI 用更系统的方式重新尝试 |

**NVC 版替代话术**：
> "这个结果和我预期的有差距（观察），我感到沮丧（感受），因为我需要这个问题被彻底解决（需要）。能否用更系统的方式重新做一版（请求）？"

## NVC 的威力

### 对用户的影响

- 把攻击性转化为澄清：从"你不行"变成"我需要什么"
- 减少防御性反应：对方不会因为被攻击而防御
- 提升沟通效率：具体的请求比模糊的攻击更容易被响应

### 对 Agent 的影响

- 理解真实需求：不被表面的攻击性分散注意力
- 提供精准帮助：知道用户真正需要什么
- 建立信任：用户感到被理解，而非被应付

---

## NVC 不是什么

### 不是软弱

NVC 不是"温柔地妥协"，而是"清晰地表达"。

"能否先搜索一下"和"你居然不搜索"传达同样的紧迫感，但前者给了对方行动的空间，后者给了对方防御的动机。

### 不是客套

NVC 不是"礼貌地绕圈子"，而是"直接地说出真相"。

"我感到沮丧，因为我需要这个问题被解决"比"你的水平很差"更直接——它直接指向了核心：你的需求。

### 不是单方面的退让

NVC 是双向的。它不是让用户单方面压抑自己的需求，而是让双方都看见彼此的需求。
